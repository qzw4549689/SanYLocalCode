## 一、模拟场景：SA级老客户（categoryId=1）

### 1. Account表（客户主数据）

在D365中修改/创建客户记录，设置以下字段：

| 字段 | 值 | 说明 |
|------|-----|------|
| `mcs_accountcategory` | **20** | SANY end customer（三一终端客户，非经销商） |
| `mcs_accountlevel` | **4** | Diamond（S级大客户） |
| `mcs_accounttype` | **2** | Company Account（公司客户） |

> 其他字段（如客户名称、国家编码等）按需填写

### 2. 销售订单表（判断新老客户）

在D365中创建至少1条销售订单，关联该客户：

| 字段 | 值 | 说明 |
|------|-----|------|
| `customerid` | [客户ID] | 关联上述Account记录 |
| `name` | "TEST-ORDER-001" | 订单名称 |

> 只要有≥1条销售订单，即判定为**老客户**

### 3. 信用评估记录

创建评估记录，关联该客户：

| 字段 | 值 | 说明 |
|------|-----|------|
| `mcs_accountid` | [客户ID] | Lookup关联Account |
| `mcs_status` | **13** | 信用分计算（触发Plugin） |

### 4. 评分卡配置表（mcs_credit_scoringcard）

需预置 categoryId=1 的评分卡配置：

| 字段 | 示例值 | 说明 |
|------|--------|------|
| `mcs_credit_scoringcardno` | "SC202506060001" | 自动生成的编码 |
| `mcs_categoryid` | **1** | SA级老客户 |
| `mcs_credititem` | [评分项目ID] | Lookup关联mcs_credit_items |
| `mcs_itemid` | "ExternalRating" | 评分项目编码（String） |
| `mcs_itemname` | "外部评级" | 评分项目名称 |
| `mcs_datatype` | **100000000** | 定量（或100000001定性） |
| `mcs_listvalue` | "6" | 定性匹配值（定性时必填） |
| `mcs_minvalue` | 1.0 | 定量最小值（定量时必填） |
| `mcs_maxvalue` | 10.0 | 定量最大值（定量时必填） |
| `mcs_weight` | 30 | 赋分（权重） |

### 5. 客户信用标签表（mcs_customer_tag）

需存在对应标签记录（含复核值）：

| 字段 | 示例值 | 说明 |
|------|--------|------|
| `mcs_credit_record` | [评估记录ID] | Lookup关联评估记录 |
| `mcs_itemid` | [评分项目ID] | Lookup关联mcs_credit_items |
| `mcs_datatype` | **1** | 定量（标签表值：1=定量, 2=定性） |
| `mcs_itemintvalue2` | 6.0 | 复核定量值（定量指标） |
| `mcs_itemtxtvalue2` | "6" | 复核定性值（定性指标） |
| `mcs_active` | **true** | 有效状态 |

## 三、完整模拟步骤（以SA级老客户为例）

```
D365路径: 客户管理 → 客户 → 打开/新建客户记录

修改字段:
  - Account Category (mcs_accountcategory) = 20 (SANY end customer)
  - Account Level (mcs_accountlevel) = 4 (Diamond)
  - Account Type (mcs_accounttype) = 2 (Company Account)
  - 客户名称 = "TEST-SA-OLD-CUSTOMER"
  - 国家编码 = "PL"
  - 客户英文名称 = "TEST SA OLD CUSTOMER"
  - Coface ID = "icon#5415240" (如需要Coface集成)
```

### Step 2: 创建销售订单（模拟老客户）

```
D365路径: 销售 → 订单 → 新建

填写字段:
  - 客户 (customerid) = 选择Step 1创建的客户
  - 订单名称 = "TEST-ORDER-001"
  - 订单日期 = 今天
  - 保存

> 只需创建1条即可判定为老客户
```

### Step 3: 预置评分卡配置

```
D365路径: 资信管理 → 客户评分卡 → 新建

同一评分项目下可创建多行分档，示例（categoryId=1）:

外部评级（定性）:
  记录1:
    - 评分卡类型 = 1 (SA级老客户)
    - 评分项目 = ExternalRating (外部评级)
    - 数据类型 = 100000001 (定性)
    - 定性项目值 = A
    - 赋分 = 20

  记录2:
    - 评分卡类型 = 1 (SA级老客户)
    - 评分项目 = ExternalRating (外部评级)
    - 数据类型 = 100000001 (定性)
    - 定性项目值 = B
    - 赋分 = 15

  记录3:
    - 评分卡类型 = 1 (SA级老客户)
    - 评分项目 = ExternalRating (外部评级)
    - 数据类型 = 100000001 (定性)
    - 定性项目值 = C
    - 赋分 = 10

  记录4（缺失值）:
    - 评分卡类型 = 1 (SA级老客户)
    - 评分项目 = ExternalRating (外部评级)
    - 数据类型 = 100000001 (定性)
    - 定性项目值 = O
    - 赋分 = 0

净资产（定量）:
  记录5:
    - 评分卡类型 = 1 (SA级老客户)
    - 评分项目 = NetAssets (净资产)
    - 数据类型 = 100000000 (定量)
    - 定量最小值 = 0
    - 定量最大值 = 1000000
    - 赋分 = 5
    - 说明: 区间 [0, 1000000)，包含0、不含1000000

  记录6:
    - 评分卡类型 = 1 (SA级老客户)
    - 评分项目 = NetAssets (净资产)
    - 数据类型 = 100000000 (定量)
    - 定量最小值 = 1000000
    - 定量最大值 = 10000000
    - 赋分 = 10
    - 说明: 区间 [1000000, 10000000)

  记录7:
    - 评分卡类型 = 1 (SA级老客户)
    - 评分项目 = NetAssets (净资产)
    - 数据类型 = 100000000 (定量)
    - 定量最小值 = 10000000
    - 定量最大值 = 99999999999.99
    - 赋分 = 20
    - 说明: 区间 [10000000, 99999999999.99)

> 定量区间语义统一为 [min, max)，即包含最小值、不包含最大值。
> 赋分允许负数；如需“无资料”得固定分，请显式配置对应分档。
> 同一 category + 评分项目 下定性值不能重复，定量区间不能重叠。
> 按PRD继续配置其他评分项目，确保总分 = 100。
```

### Step 4: 准备客户标签数据

```
方式A: 通过Coface数据集成自动创建（状态流转到11触发）
方式B: 手动创建标签记录

D365路径: 资信管理 → 客户信用标签 → 新建

创建标签记录（以净资产定量标签为例）:
  - 信用评估 = [选择Step 5创建的评估记录]
  - 评分项目 = NetAssets
  - 数据类型 = 1 (定量)
  - 集成定量指标 = 5000000.0
  - 复核定量指标 = 5000000.0
  - 有效状态 = 是

> 标签记录数量需与评分卡配置项数（评分项目数）对应；每个评分项目一条标签记录。
> 定性标签请填写复核定性指标，例如 ExternalRating = "B"。
```

### Step 5: 创建信用评估记录并触发计算

```
D365路径: 资信管理 → 客户信用评估 → 新建

填写字段:
  - 客户编码 = 选择Step 1创建的客户
  - 客户名称、英文名称、国家编码自动带出
  - 保存

触发计算:
  方式1: 修改状态字段为 13 (信用分计算) → 保存 → Plugin自动触发
  方式2: 通过BPF流程推进到"信用分计算"阶段

验证结果:
  - 状态自动变为 14 (审核申请)
  - 信用评分字段有值 (0-100)
  - 信用评分日期 = 今天
  - Plugin Trace日志显示: "匹配评分卡: 1-SA级老客户"
```

| 检查项 | 验证方法 | 预期结果 |
|--------|---------|---------|
| Account字段正确 | 查看客户表单 | category=20, level=4, type=2 |

## 五、常见问题

### Q1: 评分卡配置表为空怎么办？
**A**: 需要先手动创建评分卡配置数据。7种评分卡类型都需要配置，否则Plugin会报错"该客户还未配置评分卡"。

### Q2: 标签记录如何自动生成？
**A**: 目前Coface集成会生成12个外部指标标签。剩余内部指标标签需手动创建或通过其他方式生成（待开发）。

### Q3: 如何查看Plugin Trace日志？
**A**: 
- 方法1: D365设置 → 插件跟踪日志（Plugin Trace Log）
- 方法2: 使用Plugin Registration Tool查看日志
- 方法3: 检查评估记录的API状态/消息字段

### Q4: 状态流转不触发Plugin？
**A**: 检查以下几点：
1. Plugin是否已注册（Plugin Registration Tool）
2. 注册参数是否正确（Update/PostOp/mcs_credit_record/mcs_status）
3. 状态值是否为13（信用分计算）
4. 是否是状态字段变更触发（仅修改其他字段不会触发）

### Q5: 标签值为空或O时如何得分？
**A**: 当标签复核值为空、`N/A` 或定性缺失值 `O` 时，该评分项目会取所有分档赋分的整数平均分。如果业务希望“无/未提供资料”得固定分（如0分），请为该评分项目单独配置一行对应枚举值或区间。
