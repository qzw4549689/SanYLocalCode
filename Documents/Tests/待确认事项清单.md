## 一、评分卡匹配规则（高优先级）

### 1. 个人客户（I级）识别逻辑 ✅ 已采用方案

| 项目 | 内容 |
|------|------|
| **问题描述** | PRD定义有个人客户评分卡（I级），但D365的 `mcs_accountlevel` 选项集中无I级值 |
| **采用方案** | 使用 `mcs_accounttype` 字段（Account Type）：<br>• Value=1 → Individual Account（个人客户）<br>• Value=2 → Company Account（公司客户） |
| **当前实现** | `isPersonal = (accountType == 1)` |
| **风险** | 如业务方对个人客户的定义与D365字段含义不一致，需调整 |
| **回退方案** | 如不正确，可回退到 `category=20 && level=1` 的旧逻辑 |
| **影响范围** | CreditScorePlugin.cs 的 `MatchScoringCardType` 方法 |
| **状态** | ✅ 已采用，待业务方最终确认 |

### 2. S/A/B/C 与 Diamond/Gold/Silver/Other 的映射关系

| 项目 | 内容 |
|------|------|
| **问题描述** | PRD定义直销分级为S/A/B/C，但D365实际选项集值为Diamond/Gold/Silver/Other |
| **当前映射** | Diamond(4)=S级, Gold(3)=A级, Silver(2)=B级, Other(1)=C级 |
| **风险** | 映射关系错误会导致大客户/普通客户判断错误，进而匹配错误评分卡 |
| **需要确认** | 该映射关系是否符合业务实际定义？ |
| **影响范围** | 评分卡类型匹配逻辑、客户等级计算 |
| **状态** | ⏳ 待确认 |

### 3. 经销商分级字段 `mcs_dealerrank` 的使用

| 项目 | 内容 |
|------|------|
| **问题描述** | PRD Table 21 显示 `mcs_dealerrank`（经销商分级）字段，但当前评分卡匹配逻辑未使用该字段 |
| **当前实现** | 仅通过 `mcs_accountcategory`（10/90）判断是否为经销商，不区分经销商等级 |
| **问题** | 老经销商/新经销商评分卡是否不需要区分钻石/铂金/白银/认证/意向？ |
| **需要确认** | 经销商评分卡是否只有2套（老/新），不区分经销商等级？ |
| **影响范围** | CreditScorePlugin.cs、评分卡配置数据 |
| **状态** | ⏳ 待确认 |

## 二、数据类型值不一致（中优先级）

### 4. 各实体数据类型选项集值不统一

| 项目 | 内容 |
|------|------|
| **问题描述** | 不同实体的"数据类型"字段使用了不同的选项集值 |
| **当前状况** | `mcs_credit_scoringcard`: 100000000=定量, 100000001=定性<br>`mcs_customer_tag`: 1=定量, 2=定性<br>`mcs_credit_items`: 100000000=定量, 100000001=定性 |
| **当前处理** | 代码中做了兼容判断：`isQuantitative = (dataType == 1 \|\| dataType == 100000000)` |
| **建议** | 统一所有实体的数据类型选项集值为同一套（推荐用 100000000/100000001） |
| **影响范围** | ScoreCalculator.cs、所有涉及数据类型判断的代码 |
| **状态** | ⏳ 待确认是否需要在D365中统一选项集值 |
