## 2. D365 标准汇率现状

### 2.1 基础货币

- **DEV1 基础货币 ID**：`3bd92af1-8302-ee11-8f6e-000d3ac7739b`
- **UAT 基础货币 ID**：`c3dee6fa-c213-ee11-9cbe-6045bd1d8b49`
- **基础货币 ISO 代码**：`USD`

### 2.2 维护币种数量

| 环境 | 维护币种数 |
|---|---|
| DEV1 | 55 |
| UAT | 81 |

Coface 当前 17 种交易币种在两个环境中均已存在。

### 2.3 汇率方向

D365 `transactioncurrency.exchangerate` 的标准含义为：

```
1 Base Currency (USD) = exchangerate Local Currency
```

示例（UAT）：

- `USD = 1.0`
- `CNY = 7.0288` 表示 **1 USD = 7.0288 CNY**
- `EUR = 0.85348` 表示 **1 USD = 0.85348 EUR**

而 Coface 自定义表 `mcs_coface_exchange_rate.mcs_rate_to_usd` 的含义为：

```
1 Local Currency = mcs_rate_to_usd USD
```

示例：

- `CNY = 0.1463` 表示 **1 CNY = 0.1463 USD**
- 等价于 **1 USD ≈ 6.8353 CNY**

因此，若改用 D365 汇率，需要在代码中对非 USD 币种取倒数：

```csharp
var rateToUsd = 1m / d365ExchangeRate;
```

## 4. 风险与影响

### 4.1 业务规则冲突

Coface 项目文档明确使用 **2026 Budget 固定汇率**，而 D365 标准汇率通常用于交易/实时换算。若 D365 汇率被财务部门定期更新，Coface 的年度固定口径将被打破。

### 4.2 换算方向与精度

- 需对所有非 USD 币种执行 `1 / exchangerate`。
- 对于 IDR、VND 等小币种，D365 汇率值很大（万级），取倒数后会产生极小数值（`0.0000x`），需确认目标字段精度是否足够。

### 4.3 全局影响面

D365 `transactioncurrency` 是组织级配置，被以下模块共享：

- 销售报价 / 订单 / 发票
- 回款 / 财务集成
- 客户 / 产品货币字段
- 报表与 KPI

仅因 Coface 需要而调整 D365 汇率，可能影响全组织财务数据。

### 4.4 环境一致性

DEV1 与 UAT 的汇率数据差异显著（同币种如 EUR：DEV1 0.91 vs UAT 0.85），说明缺少统一维护流程。上线前需确保 DEV/UAT/PRD 三环境一致。

## 5. 待确认事项清单

在继续推进前，需要业务/财务/架构方书面确认以下问题：

1. **汇率性质**：D365 标准汇率是否代表 2026 Budget 固定汇率？还是实时/交易汇率？
2. **维护责任**：谁负责维护 D365 中的汇率？更新频率是多久？
3. **汇率方向**：Coface 代码需要 `1 LC → USD`，是否接受对 D365 汇率取倒数？
4. **数据清洗**：DEV1 中 KRW/MYR/SGD/INR/VND 等异常/缺失数据如何修正？
5. **环境同步**：DEV1/UAT/PRD 的汇率是否已保持一致？如何验证？
6. **影响范围**：更改汇率后，是否需要重新跑历史 Coface 评分/额度数据？
7. **回滚策略**：若上线后发现问题，是否保留回退到 `mcs_coface_exchange_rate` 的能力？

## 6. 建议方案

### 6.1 短期（当前阶段）

- **暂停代码替换**，保留现有 `mcs_coface_exchange_rate` 方案。
- 由财务部门提供 D365 汇率维护说明及与 Coface 2026 Budget 汇率的差异说明。
- 若业务坚持使用 D365 汇率，**先在 DEV1 清洗异常汇率数据**，并补齐 VND。

### 6.2 中期（确认可行后）

若确认改用 D365 汇率，建议采用**双轨读取**策略：

1. 新增 Plugin 配置项（如 `UseD365ExchangeRate`），默认关闭；
2. `CofaceExchangeRateHelper.GetRateToUsd` 先判断配置：
   - 若启用，从 `transactioncurrency` 读取 `exchangerate` 并取倒数；
   - 若未启用，仍从 `mcs_coface_exchange_rate` 读取；
3. 上线后小范围验证，确认无偏差再全量切换。

这样可保留回滚能力，符合项目对 Coface 汇率“独立可控”的设计初衷。
