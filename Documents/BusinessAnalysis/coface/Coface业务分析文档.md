### 1.3 Coface数据集成详细流程

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                         Coface数据集成流程 ( per客户 )                        │
└─────────────────────────────────────────────────────────────────────────────┘

Phase 1: 企业匹配 (新客户)
─────────────────────────
CRM ──GET /companies──> Coface ICON
  入参: name(英文名称), countryCode(ISO-2), range=1, limit=20
  
Coface ICON ──200/206──> CRM
  出参: companies[].externalIds[] (取repositorySlug="icon"的id)
        拼接为: "icon#" + id
        同时获取: taxNumber, registrationNumber, address, city
  
CRM: 人工选择匹配企业 → 保存externalId到客户表


Phase 2: 订单状态检查 (URBA360 Monitoring)
───────────────────────────────────────────
CRM ──GET /urba360/monitorings/orders──> Coface ICON
  入参: externalId, countryCode
  
Coface ICON ──200──> CRM
  出参: orders[] { id, startDate, endDate, urbaStatus }
  
CRM判断:
  ├─ 存在有效订单(urbaStatus=ready) → 跳过下单，直接获取数据
  ├─ 订单不存在/过期 → 需下单 (开发阶段关闭此功能，采用线下预处理)
  └─ 订单准备中(urbaStatus=in_progress) → 提示等待6-7工作日


Phase 3: 获取URBA360指标数据 (9个指标)
───────────────────────────────────────
CRM ──GET /urba360/content──> Coface ICON
  入参: id (Order ID)
  
Coface ICON ──200──> CRM
  出参: 
    ├─ 外部评级: debtorRiskValue (取isCurrent=true或newUpdate=true)
    │           枚举值1-10
    ├─ 迟付指数: latePaymentIndex.value (取newUpdate=true)
    │           值域0-1，表示百分比(如0.10=10%)
    ├─ 客户财务(4个):
    │   ├─ 净资产: financials.balanceSheet.balanceSheetItems["Net Assets"]
    │   │         或 ["Net Worth"] 或 ["Equity"]
    │   ├─ 资产负债率: financials.ratios["Debt ratio"].resultValue
    │   ├─ 流动比率: financials.ratios["Current Ratio"].resultValue
    │   └─ 净利润率: financials.ratios["Net Profit Margin"].resultValue
    │   金额类型: number, 4位小数, 货币USD
    ├─ 国别风险: countryRiskAssessment.countryRiskValue (文本值)
    ├─ 行业风险: sectorRiskAssessment.sectorRiskValue (文本值)
    └─ 行业属性: naceCodes[] (文本值)


Phase 4: 获取Report指标数据 (3个指标)
─────────────────────────────────────
CRM ──GET /publications/orders──> Coface ICON
  入参: externalId, countryCode
  
CRM判断: 存在有效report订单? 
  ├─ 是 → 获取数据
  └─ 否 → 提示需下单 (开发阶段关闭)

CRM ──GET /publications──> Coface ICON
  入参: id (publications.id), format="json"
  
Coface ICON ──200──> CRM
  出参:
    ├─ 注册资本: icon.creditReport.shareCapital
    │   取: max(nominalCapitalAmount, issuedCapitalAmount, paidUpCapitalAmount)
    │   本地货币 → 需转换为USD
    │   ⚠️ 注意: 单位是千非元，需×1000
    ├─ 从业年限: icon.creditReport.company.registration.date
    │   格式: YYYYMMDD，计算到当前日期的年数
    └─ 诉讼债权标的金额: PDF人工判断 (5/15确认)
        接口获取PDF → 人工阅读 → 手工录入
        (原JSON路径因数据字典不完整，无法自动提取)


Phase 5: 数据转换与存储
───────────────────────
CRM处理:
  ├─ 货币转换: Coface本地货币 → USD (注册资本等)
  ├─ 单位转换: 千 → 元 (注册资本需×1000)
  ├─ 缺失值处理:
  │   ├─ 定量指标缺失 → 值=-1
  │   └─ 定性指标缺失 → 值='O'
  └─ 保存到: 客户信用标签表 (mcs_customer_tag)
```

## 五、关键风险与注意事项

| 风险 | 影响 | 缓解措施 |
