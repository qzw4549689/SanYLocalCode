2. 确认需要同时携带 `x-api-key` Header

4. 确认每次数据请求必须同时带 `Authorization: Bearer {idToken}` + `x-api-key: {API_KEY}`

**认证流程**:
```
POST /authentication/v1/token
  Headers: x-api-key, Content-Type
  Body: {username, password, grant_type: "password"}

```

## 关键发现与经验总结

### 1. 认证方式
- Coface 使用 AWS Cognito 认证
- `idToken` 用于数据接口（不是 `accessToken`）
- 必须同时带 `Authorization: Bearer {idToken}` + `x-api-key: {API_KEY}`
- Token 有效期 3600 秒

### 2. URBA360 内容获取
- **正确接口**: `GET /urba360/content?id={orderId}`
- 不是 `/urba360/orders/{id}/publications/{pubId}`（返回403）
- 返回完整 URBA360 数据，包含所有子模块

### 3. Full Report 内容获取
- **正确接口**: `GET /publications?id={publicationId}&format=json`
- `publicationId` 从 `GET /publications/orders` 响应中获取
- 注意区分 `orderId` 和 `publicationId`

### 4. 数据特点
- URBA360 和 Full Report 有数据重叠
- 财务数据在 URBA360 的 `productDetails.financials` 中
- 诉讼记录只在 Full Report 中提供
- 需注意货币转换（EUR → USD）和 dimension 单位

## 遇到的问题与解决

| 问题 | 解决方式 |
|------|---------|
| 403 Forbidden | 同时携带 idToken + x-api-key |
| 401 Unauthorized | 使用 idToken 而非 accessToken |
| 端点路径错误 | 查阅文档确认正确路径 |
| 数据缺失（注册资本/诉讼） | 确认需从 Full Report 获取 |
