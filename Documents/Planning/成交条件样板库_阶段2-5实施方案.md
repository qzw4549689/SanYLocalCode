### 2.2 实现 `mcs_tradetermid` 自动编码规则 Plugin

#### Plugin 设计

| 项目 | 内容 |
|---|---|
| 类名 | `TradeStPayTermAutoNumberPlugin` |
| 命名空间 | `SanyD365.D365Extension.Sales.Plugins.TradeStPayTerm`（远程） / 本地独立命名空间 |
| 触发实体 | `mcs_trade_stpayterm` |
| 触发事件 | Create / PreOperation |
| 功能 | 生成标准条件编码，赋值给主字段 `mcs_trade_stpaytermname` |

#### 编码规则（暂定）

```
TC + YYMMDD + 2位序号
示例：TC26062101
```

> 注：因主字段长度限制为 10，`TC` + `YYMMDD`（6 位） + 2 位序号 = 10 位。如业务需要更多位数或不同前缀，需调整。

#### 生成逻辑

1. 校验 `target.LogicalName == "mcs_trade_stpayterm"`
2. 如果主字段已有值，直接返回
3. 查询当天已创建的最大编码：
   - FetchXML 过滤 `mcs_trade_stpaytermname` Like `TC{yyymmdd}%`
   - 按编码降序取第一条
4. 新序号 = 最大序号 + 1，不足两位补零
5. 赋值给 `mcs_trade_stpaytermname`

### 2.4 实现保存校验 Plugin

#### Plugin 设计

| 项目 | 内容 |
|---|---|
| 类名 | `TradeStPayTermValidationPlugin` |
| 触发实体 | `mcs_trade_stpayterm` |
| 触发事件 | Create / Update / PreOperation |
| 功能 | 保存前业务规则校验 |

#### 校验规则

1. **首付款比例校验**
   - `mcs_downpay` 必须在 [0, 1] 之间
   - 精确到小数两位

2. **账期/付款频次校验**
   - `mcs_payterm` 和 `mcs_payfreq` 必须是 0 或 30 的倍数
   - 值域 [0, 9999]

3. **100% 首付款一致性校验**
   - 如果 `mcs_downpay` = 1，则 `mcs_payterm` 必须 = 0 且 `mcs_payfreq` 必须 = 0

4. **重复记录校验**
   - 维度：事业部编码、子公司编码、国家代码、产品分类编码、客户分类代码
   - 子公司/国家/产品分类为空或 NA 时，视为通配
   - 多选字段解析后取交集，如存在交集则视为重复
   - 抛出 `InvalidPluginExecutionException("存在有重复记录，需核查！")`

#### 重复校验算法思路

```csharp
// 1. 构建当前记录的条件集合
var currentSubId = string.IsNullOrEmpty(subId) || subId == "NA" ? null : subId;
var currentCountries = ParseMultiSelect(countryCode, ","); // 空或 NA 视为 null
var currentTypes = ParseMultiSelect(typeId, ",");          // 空或 NA 视为 null
var currentBuyerGrades = ParseMultiSelect(buyerGrade, "/");

// 2. 查询同事业部的其他记录
var query = new QueryExpression("mcs_trade_stpayterm")
{
    ColumnSet = new ColumnSet("mcs_subid", "mcs_countrycode", "mcs_typeid", "mcs_buyergrade"),
    Criteria = new FilterExpression
    {
        Conditions =
        {
            new ConditionExpression("mcs_buid", ConditionOperator.Equal, buId),
            new ConditionExpression("mcs_trade_stpaytermid", ConditionOperator.NotEqual, currentId)
        }
    }
};

// 3. 逐条判断是否存在冲突
foreach (var record in records)
{
    if (IsConflict(currentSubId, currentCountries, currentTypes, currentBuyerGrades, record))
    {
        throw new InvalidPluginExecutionException("存在有重复记录，需核查！");
    }
}
```

### 2.5 实现【克隆新增】功能

#### 实现方式

#### 新增文件

**JS 文件**：`Code/Customizations/WebResources/JS/mcs_trade_stpayterm.js`

```javascript
var TradeStPayTermForm = TradeStPayTermForm || {};

TradeStPayTermForm.onLoad = function (executionContext) {
    var formContext = executionContext.getFormContext();
    var cloneId = TradeStPayTermForm.getQueryParam("cloneId");
    
    if (cloneId) {
        TradeStPayTermForm.fillFromClone(formContext, cloneId);
    }
};

TradeStPayTermForm.fillFromClone = function (formContext, cloneId) {
    Xrm.WebApi.retrieveRecord("mcs_trade_stpayterm", cloneId,
        "?$select=mcs_buid,mcs_buname,mcs_subid,mcs_subname,mcs_countrycode,mcs_countryname,mcs_typeid,mcs_typename,mcs_buyergrade,mcs_creditgrade,mcs_downpay,mcs_payterm,mcs_payfreq")
        .then(function (result) {
            for (var field in result) {
                if (formContext.getAttribute(field)) {
                    formContext.getAttribute(field).setValue(result[field]);
                }
            }
            // 清空主字段，让自动编码 Plugin 重新生成
            if (formContext.getAttribute("mcs_trade_stpaytermname")) {
                formContext.getAttribute("mcs_trade_stpaytermname").setValue(null);
            }
            // 状态重置为未生效
            if (formContext.getAttribute("mcs_status")) {
                formContext.getAttribute("mcs_status").setValue(0);
            }
        });
};

TradeStPayTermForm.getQueryParam = function (name) {
    var regex = new RegExp("[?&]" + name + "(=([^&#]*)|&|#|$)");
    var results = regex.exec(window.location.href);
    if (!results || !results[2]) return null;
    return decodeURIComponent(results[2].replace(/\+/g, " "));
};
```

TradeStPayTermGrid.apply = function (primaryControl) {
    var selected = primaryControl.getSelectedRecordIds();
    if (selected.length === 0) {
        Xrm.Navigation.openAlertDialog({ text: "请至少选择一条记录。" });
        return;
    }
    
    Xrm.Navigation.openConfirmDialog({
        title: "确认申请",
        text: "确定要将选中的 " + selected.length + " 条记录提交申请吗？"
    }).then(function (success) {
        if (!success.confirmed) return;
        
        var promises = selected.map(function (id) {
            return Xrm.WebApi.updateRecord("mcs_trade_stpayterm", id, {
                "mcs_status": 1
            });
        });
        
        Promise.all(promises).then(function () {
            Xrm.Navigation.openAlertDialog({ text: "申请提交成功。" });
            primaryControl.refresh();
        }).catch(function (error) {
            Xrm.Navigation.openAlertDialog({ text: "申请失败：" + error.message });
        });
    });
};
```

// 只有状态 0（未生效）的记录才能更新为 1（待审批）
var oldStatus = preImage.GetAttributeValue<OptionSetValue>("mcs_status")?.Value ?? 0;

if (oldStatus == 0 && newStatus == 1) { /* 允许 */ }

TradeStPayTermGrid.approve = function (primaryControl) {
    TradeStPayTermGrid.batchUpdateStatus(primaryControl, 2, "审批");
};

TradeStPayTermGrid.reject = function (primaryControl) {
    TradeStPayTermGrid.batchUpdateStatus(primaryControl, 0, "拒绝");
};

TradeStPayTermGrid.batchUpdateStatus = function (primaryControl, status, actionName) {
    var selected = primaryControl.getSelectedRecordIds();
    if (selected.length === 0) {
        Xrm.Navigation.openAlertDialog({ text: "请至少选择一条记录。" });
        return;
    }
    
    Xrm.Navigation.openConfirmDialog({
        title: "确认" + actionName,
        text: "确定要" + actionName + "选中的 " + selected.length + " 条记录吗？"
    }).then(function (success) {
        if (!success.confirmed) return;
        
        var promises = selected.map(function (id) {
            return Xrm.WebApi.updateRecord("mcs_trade_stpayterm", id, {
                "mcs_status": status
            });
        });
        
        Promise.all(promises).then(function () {
            Xrm.Navigation.openAlertDialog({ text: actionName + "成功。" });
            primaryControl.refresh();
        }).catch(function (error) {
            Xrm.Navigation.openAlertDialog({ text: actionName + "失败：" + error.message });
        });
    });
};
```

- 【审批】/【拒绝】按钮仅在"待审批"视图显示

可通过 Command Bar 的 `visibilitytype` 或 JS 动态控制。D365 Modern Command Bar 支持基于视图的显示规则，但较复杂。MVP 阶段可先不做严格视图绑定，靠后端 Plugin 校验阻止非法操作。

## 阶段 4：成交条件计算公共接口

### 4.1 客户分类/客户等级计算逻辑

#### 客户分类计算

**输入**：客户编码
**输出**：客户分类代码（S/A/B/C/I/D1/D2/D3/D4/D5）

**逻辑步骤**：

1. 查询 `mcs_customermasterdata`，获取字段：
   - `mcs_customertype`（客户类型：公司/个人）
   - `mcs_accountcategory`（客户类别）
   - `mcs_accountlevel`（大客户级别）
   - `mcs_dealerrank`（经销商分级）

2. 判断是否经销商：
   - 客户类别包含"经销商"字样，或经销商分级有值

3. 经销商分类映射：
   - 钻石 → D1
   - 铂金 → D2
   - 白银 → D3
   - 认证 → D4
   - 意向 → D5

4. 直销客户分类：
   - 大客户级别 = S/A/B → 对应 S/A/B
   - 客户类型 = 个人 → I
   - 其他 → C

> **待确认**：客户主数据表上客户类别、大客户级别、经销商分级的实际字段逻辑名，需要结合现有 CRM 字段确认。

#### 客户等级计算

客户等级 `mcs_creditgrade` 从客户主数据表读取（A0-A4），本阶段暂不参与查询，但计算逻辑保留。

### 4.3 实现成交条件样板库查询 API

#### 接口形态对比

| 方案 | 实现位置 | 优点 | 缺点 | 适用场景 |
|---|---|---|---|---|
| A. Custom API | D365 Plugin | 部署简单，D365 内直接调用，无需 Azure 发布 | 受 Plugin 执行时间限制 | D365 内部模块调用 |
| B. Inner API | Service 项目 | 可处理复杂逻辑和大数据量，适合外部系统 | 需要 Azure 部署，n8n 勾选 InnerAPI | 外部系统或大量并发 |

> **待确认**：请选择方案 A 或 B。

#### 方案 A：Custom API 设计

**Custom API 唯一名**：`mcs_QueryTradeStPayTerm`

**输入参数**：

| 参数名 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `mcs_buid` | 文本 | 是 | 事业部编码 |
| `mcs_subid` | 文本 | 是 | 子公司编码 |
| `mcs_countrycode` | 文本 | 是 | 国家代码 |
| `mcs_prdgroupid` | 文本 | 是 | 产品线编码 |
| `mcs_buyercode` | 文本 | 是 | 客户编码 |

**输出参数**：

| 参数名 | 类型 | 说明 |
|---|---|---|
| `status` | 文本 | 1-成功，0-错误 |
| `message` | 文本 | 错误信息 |
| `records` | 文本（JSON 字符串） | 匹配记录列表 |

**处理逻辑**：

1. 校验入参非空
2. 根据客户编码计算客户分类
3. 根据产品线编码查询产品分类编码
4. 判断事业部是否为泵路事业部
5. 构建 FetchXML 查询 `mcs_trade_stpayterm`
6. 返回匹配记录

#### 方案 B：Inner API 设计

**Controller**：`TradeStPayTermController`

**Action**：`POST /api/tradestpayterm/query`

**请求/响应**：同方案 A 的输入输出，返回 JSON。

**部署**：n8n Release Tool 勾选 `InnerAPI`。

### 4.4 查询条件构建

#### 泵路事业部

```csharp
var query = new QueryExpression("mcs_trade_stpayterm")
{
    ColumnSet = new ColumnSet("mcs_trade_stpaytermname", "mcs_buid", "mcs_buname", "mcs_subid", "mcs_subname",
        "mcs_countrycode", "mcs_countryname", "mcs_typeid", "mcs_typename", "mcs_buyergrade",
        "mcs_creditgrade", "mcs_downpay", "mcs_payterm", "mcs_payfreq"),
    Criteria = new FilterExpression
    {
        Conditions =
        {
            new ConditionExpression("mcs_buid", ConditionOperator.Equal, buId),
            new ConditionExpression("mcs_subid", ConditionOperator.Equal, subId),
            new ConditionExpression("mcs_status", ConditionOperator.Equal, 2)
        }
    }
};

// 产品分类包含匹配
query.Criteria.AddCondition("mcs_typeid", ConditionOperator.Like, $"%{typeId}%");

// 客户分类包含匹配
query.Criteria.AddCondition("mcs_buyergrade", ConditionOperator.Like, $"%{buyerGrade}%");
```

#### 其他事业部

```csharp
var filter = new FilterExpression(LogicalOperator.And);
filter.Conditions.Add(new ConditionExpression("mcs_buid", ConditionOperator.Equal, buId));
filter.Conditions.Add(new ConditionExpression("mcs_status", ConditionOperator.Equal, 2));

// 子公司 = 入参 OR 子公司 = NA
var subFilter = new FilterExpression(LogicalOperator.Or);
subFilter.Conditions.Add(new ConditionExpression("mcs_subid", ConditionOperator.Equal, subId));
subFilter.Conditions.Add(new ConditionExpression("mcs_subid", ConditionOperator.Equal, "NA"));
filter.AddFilter(subFilter);

// 国家包含入参 OR 国家 = NA
var countryFilter = new FilterExpression(LogicalOperator.Or);
countryFilter.Conditions.Add(new ConditionExpression("mcs_countrycode", ConditionOperator.Like, $"%{countryCode}%"));
countryFilter.Conditions.Add(new ConditionExpression("mcs_countrycode", ConditionOperator.Equal, "NA"));
filter.AddFilter(countryFilter);

// 产品分类包含入参 OR 产品分类 = NA
var typeFilter = new FilterExpression(LogicalOperator.Or);
typeFilter.Conditions.Add(new ConditionExpression("mcs_typeid", ConditionOperator.Like, $"%{typeId}%"));
typeFilter.Conditions.Add(new ConditionExpression("mcs_typeid", ConditionOperator.Equal, "NA"));
filter.AddFilter(typeFilter);

// 客户分类包含入参
filter.Conditions.Add(new ConditionExpression("mcs_buyergrade", ConditionOperator.Like, $"%{buyerGrade}%"));

query.Criteria = filter;
```

> **注意**：Like 查询在大量数据下性能可能不佳。如数据量大，需要考虑拆分多选字段为子表或采用其他索引策略。
