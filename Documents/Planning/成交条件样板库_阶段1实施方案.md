## 1. 创建 `mcs_trade_pttype` 实体及字段

### 1.1 操作方式

使用 MetadataTool 的 `create` 命令，从 JSON 定义文件创建实体和字段。

### 1.2 新增文件

**文件路径**：`Code/Tools/MetadataTool/Definitions/mcs_trade_pttype.json`

```json
{
  "entityName": "mcs_trade_pttype",
  "displayName": "成交条件产品分类",
  "primaryAttribute": "mcs_trade_pttypename",
  "primaryAttributeDisplayName": "成交条件产品分类名称",
  "primaryAttributeLength": 100,
  "solutionName": "entity_20260603_peter",
  "autoPublish": true,
  "fields": [
    {
      "schemaName": "mcs_typeid",
      "displayName": "产品分类编码",
      "description": "业务唯一编码，成交条件产品分类主关键字",
      "type": "string",
      "required": true,
      "maxLength": 10
    },
    {
      "schemaName": "mcs_typenameen",
      "displayName": "产品分类英文名称",
      "description": "产品分类英文名称",
      "type": "string",
      "required": false,
      "maxLength": 150
    }
  ]
}
```

### 1.3 设计说明

| 字段 | 来源 | 说明 |
|---|---|---|
| `mcs_trade_pttypename` | D365 主字段 | 复用为主字段，对应 PRD 中的 `mcs_typename`（中文名称），避免冗余 |
| `mcs_typeid` | PRD | 业务编码，必填，长度 10 |
| `mcs_typenameen` | PRD | 英文名称，可选 |

> **说明**：D365 实体必须有一个主字段（Primary Name Attribute）。本方案将主字段直接作为中文名称使用，与 PRD 字段语义对齐，不额外创建 `mcs_typename` 文本字段。

### 1.4 执行命令

```bash
cd Code/Tools/MetadataTool
dotnet run create Definitions/mcs_trade_pttype.json
```

### 1.5 验收标准

- [ ] DEV1 中实体 `mcs_trade_pttype` 已创建
- [ ] 字段 `mcs_typeid`、`mcs_typenameen` 已创建
- [ ] 主字段 `mcs_trade_pttypename` 显示正常
- [ ] 实体已加入解决方案 `entity_20260621_peter`
- [ ] 实体已发布

## 2. 创建 `mcs_trade_ptgrouptype` 实体及字段、Lookup 关系

### 2.1 操作方式

使用 MetadataTool 的 `create` 命令。由于包含 Lookup 字段，需确保 `mcs_trade_pttype` 已先创建。

### 2.2 新增文件

**文件路径**：`Code/Tools/MetadataTool/Definitions/mcs_trade_ptgrouptype.json`

```json
{
  "entityName": "mcs_trade_ptgrouptype",
  "displayName": "成交条件产品分类关系",
  "primaryAttribute": "mcs_trade_ptgrouptypename",
  "primaryAttributeDisplayName": "成交条件产品分类关系名称",
  "primaryAttributeLength": 100,
  "solutionName": "entity_20260603_peter",
  "autoPublish": true,
  "fields": [
    {
      "schemaName": "mcs_groupid",
      "displayName": "产品线编码",
      "description": "产品线编码，组合关键字之一",
      "type": "string",
      "required": true,
      "maxLength": 10
    },
    {
      "schemaName": "mcs_groupname",
      "displayName": "产品线名称",
      "description": "产品线名称",
      "type": "string",
      "required": true,
      "maxLength": 100
    },
    {
      "schemaName": "mcs_typeid",
      "displayName": "产品分类编码",
      "description": "成交条件产品分类编码，组合关键字之一",
      "type": "string",
      "required": true,
      "maxLength": 10
    },
    {
      "schemaName": "mcs_typename",
      "displayName": "产品分类名称",
      "description": "成交条件产品分类名称",
      "type": "string",
      "required": true,
      "maxLength": 100
    },
    {
      "schemaName": "mcs_trade_pttypeid",
      "displayName": "成交条件产品分类",
      "description": "关联成交条件产品分类",
      "type": "lookup",
      "required": true,
      "targetEntity": "mcs_trade_pttype",
      "targetEntityDisplayName": "成交条件产品分类"
    }
  ]
}
```

### 2.3 设计说明

| 字段 | 类型 | 说明 |
|---|---|---|
| `mcs_groupid` | 文本 | 产品线编码，来自基础数据产品线表 |
| `mcs_groupname` | 文本 | 产品线名称，冗余存储用于显示 |
| `mcs_typeid` | 文本 | 产品分类编码，冗余存储用于查询 |
| `mcs_typename` | 文本 | 产品分类名称，冗余存储用于显示 |
| `mcs_trade_pttypeid` | Lookup | 关联 `mcs_trade_pttype`，建立 N:1 关系 |

> **关系说明**：一个成交条件产品分类包含多条产品线关系（1:N），一条产品线关系属于一个产品分类（N:1）。

### 2.4 执行命令

```bash
cd Code/Tools/MetadataTool
dotnet run create Definitions/mcs_trade_ptgrouptype.json
```

### 2.5 验收标准

- [ ] 实体 `mcs_trade_ptgrouptype` 已创建
- [ ] Lookup 字段 `mcs_trade_pttypeid` 已创建，并正确关联 `mcs_trade_pttype`
- [ ] 1:N 关系 `mcs_trade_pttype_mcs_trade_ptgrouptype_mcs_trade_pttypeid` 已建立
- [ ] 实体已加入解决方案并发布

## 4. Excel 模板导出/导入方案

### 4.1 方案选择

**MVP 阶段建议：复用 D365 原生导入导出能力。**

理由：
- PRD 要求的基础 CRUD + Excel 导入导出，D365 列表视图的"导出到 Excel"和"从 Excel 导入数据"已能满足
- 自定义 Excel 模板开发成本较高，且在产品分类数据量不大的场景下收益有限
- 如果后续需要带校验、示例数据、批量模板的自定义导入，可在阶段 2 之后再增强

### 4.2 操作方式

**导出模板**：
1. 进入 `mcs_trade_pttype` 列表视图
2. 点击 **导出 > 导出至 Excel（静态工作表）**
3. 删除数据行，保留标题行，作为导入模板

**导入数据**：
1. 准备好 Excel 文件
2. 进入 `mcs_trade_pttype` 列表视图
3. 点击 **导入数据 > 从 Excel 导入**
4. 按向导完成字段映射和导入

### 4.3 自定义导入模板（后续可选）

如需要自定义导入模板，可后续新增 MetadataTool 命令，例如：

```bash
dotnet run export-import-template mcs_trade_pttype /tmp/mcs_trade_pttype_template.xlsx
```

此功能不在本阶段实现。

### 4.4 验收标准

- [ ] 能从列表视图导出 Excel
- [ ] 能用 Excel 批量导入产品分类数据
- [ ] 导入后数据能在视图中正确显示
