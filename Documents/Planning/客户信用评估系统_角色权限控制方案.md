# 客户信用评估系统 — 角色权限控制方案

> 来源：PRD 2.1.5 角色权限设计
> 适用范围：`mcs_credit_record`、`mcs_credit_scoringcard`、`mcs_trade_stpayterm`、`account` 等模块
> 更新日期：2026-06-23

---

## 一、角色说明

| 角色 | 数据范围 | 可执行操作 | 系统定位 |
|---|---|---|---|
| 营销人员 | 本人 | 发起信用评估、内外部数据集成、人工补入和复核 | 业务录入方 |
| 事业部风控人员 | 本部门及下级部门 | 发起信用评估、内外部数据集成、人工补入和复核 | 部门风控复核方 |
| 总部风控人员 | 全集团 | 对客户信用评分结果发布进行审批 | 最终审批方 |

> 说明：BPP 审批流程不在本方案范围内，总部风控人员在 BPP 平台完成审批操作。

## 二、按钮/控件权限控制总表

| 按钮/控件 | 所在实体/位置 | 当前控制 | 目标角色权限 | 控制方案 | 实现方式 |
|---|---|---|---|---|---|
| **新建 / 发起信用评估** | `mcs_credit_record` 列表 + 表单 | 无 | 营销人员、事业部风控可创建；总部风控只读查看 | 安全角色控制 Create 权限 | D365 安全角色：营销/事业部风控给 `mcs_credit_record` Create；总部风控不给 Create |
| **数据集成刷新** | `mcs_credit_record` 表单命令栏 | 状态 10/11 显示 | 营销人员、事业部风控可用；总部风控隐藏 | 角色 + 状态双重控制 | JS Enable Rule：`isBusinessUser() && isStatusIn([10,11])` |
| **重新发起** | `mcs_credit_record` 表单命令栏 | 无 | 营销人员、事业部风控可用；总部风控隐藏 | 角色控制 | JS Enable Rule：`isBusinessUser()` |
| **搜索 Coface 企业** | `mcs_credit_record` 表单命令栏 | 状态 9/10 + `mcs_cofaceid` 为空 | 营销人员、事业部风控可用；总部风控隐藏 | 角色 + 状态 + 字段值三重控制 | JS Enable Rule：`isBusinessUser() && isStatusIn([9,10]) && isBlank(cofaceid)` |
| **下一步 / 提交审批** | `mcs_credit_record` 状态条/命令栏 | 状态驱动 | 营销人员、事业部风控可用；总部风控隐藏 | 角色 + 状态控制 | JS Enable Rule：`isBusinessUser() && isStatusIn([9,10,11,12,13])` |
| **克隆新建** | `mcs_credit_scoringcard` 表单 | 无 | 事业部风控、总部风控可用；营销人员隐藏 | 角色控制 | JS Enable Rule：`isRiskUser()`（事业部风控 + 总部风控） |
| **批量申请** | `mcs_trade_stpayterm` 列表命令栏 | 勾选后显示 | 营销人员、事业部风控可用；总部风控隐藏 | 角色 + 选中记录控制 | Power Fx Visibility：`CountRows(Self.Selected.AllItems) > 0 && isBusinessUser()` |
| **批量审批** | `mcs_trade_stpayterm` 列表命令栏 | 勾选后显示 | 总部风控可用 | 角色 + 选中记录控制 | Power Fx Visibility：`CountRows(Self.Selected.AllItems) > 0 && isApprover()` |
| **批量拒绝** | `mcs_trade_stpayterm` 列表命令栏 | 勾选后显示 | 总部风控可用 | 角色 + 选中记录控制 | Power Fx Visibility：`CountRows(Self.Selected.AllItems) > 0 && isApprover()` |
| **克隆新增** | `mcs_trade_stpayterm` 表单 | 无 | 营销人员、事业部风控可用；总部风控可查看但隐藏按钮 | 角色控制 | JS Enable Rule：`isBusinessUser()` |
| **人工复核字段编辑** | `mcs_credit_record` 表单字段 | 状态 12 可编辑 | 营销人员、事业部风控在状态 12 可编辑；总部风控只读 | 角色 + 状态控制 | JS `setDisabled`：`isBusinessUser() && status === 12 ? false : true` |
| **信用分/等级字段** | `mcs_credit_record` 表单 | 已只读 | 全部只读 | 系统计算，禁止人工修改 | 字段属性设为只读 |
| **客户信用画像页签** | `account` 表单 | 无 | 三角色均可查看 | 默认查看权限 | 暂不做限制，后续按需 |

## 三、JS 辅助函数（建议放入 `mcs_credit_record.js` 和 `mcs_trade_stpayterm.js`）

```javascript
/**
 * 判断当前用户是否为业务人员（营销人员 / 事业部风控人员）
 */
function isBusinessUser() {
    var roles = Xrm.Utility.getGlobalContext().userSettings.roles;
    return roles.some(function(r) { 
        return r.name === "信用评估_营销人员" || r.name === "信用评估_事业部风控人员"; 
    });
}

/**
 * 判断当前用户是否为总部风控人员
 */
function isApprover() {
    var roles = Xrm.Utility.getGlobalContext().userSettings.roles;
    return roles.some(function(r) { return r.name === "信用评估_总部风控人员"; });
}

/**
 * 判断当前用户是否为风控人员（事业部风控 + 总部风控）
 */
function isRiskUser() {
    var roles = Xrm.Utility.getGlobalContext().userSettings.roles;
    return roles.some(function(r) { 
        return r.name === "信用评估_事业部风控人员" || r.name === "信用评估_总部风控人员"; 
    });
}

/**
 * 判断当前记录状态是否在指定范围内
 */
function isStatusIn(statusArray) {
    var status = Xrm.Page.getAttribute("mcs_status").getValue();
    return statusArray.indexOf(status) >= 0;
}
```

## 四、安全角色基础权限建议

| 实体 | 营销人员 | 事业部风控人员 | 总部风控人员 |
|---|---|---|---|
| `mcs_credit_record` | 读/写/创建 = 用户 | 读/写/创建 = 业务部门及子部门 | 读 = 组织，写 = 组织 |
| `mcs_credit_scoringcard` | 读 = 组织 | 读/写/创建 = 业务部门及子部门 | 读/写/创建 = 组织 |
| `mcs_customer_tag` | 读/写 = 用户 | 读/写 = 业务部门及子部门 | 读 = 组织 |
| `mcs_trade_stpayterm` | 读/写/创建 = 用户 | 读/写/创建 = 业务部门及子部门 | 读 = 组织 |
| `account` | 读 = 用户 | 读 = 业务部门及子部门 | 读 = 组织 |

> 说明："用户"指只能操作 owner 为自己的记录；"业务部门及子部门"指 owner 在本部门或下级部门的记录；"组织"指全部记录。

## 五、实施步骤

1. **创建安全角色**
   - 在 D365 中创建 3 个安全角色：`信用评估_营销人员`、`信用评估_事业部风控人员`、`信用评估_总部风控人员`
   - 按第 4 节配置各实体的读/写/创建权限

2. **配置业务部门层级**
   - 将用户分配到正确的业务部门
   - 确保事业部风控人员所在部门包含下级部门结构

3. **修改 JS 按钮控制**
   - 在 `mcs_credit_record.js` 中增加 `isBusinessUser` / `isApprover` / `isRiskUser` 函数
   - 在 `mcs_trade_stpayterm.js` 中复用上述函数
   - 为各命令栏按钮添加 Enable Rule 或 Visible 控制

4. **配置字段级安全**
   - 信用分、等级等系统计算字段保持只读
   - 复核字段在状态 12 由业务人员编辑

5. **服务端兜底校验**
   - 在状态变更 Plugin 中增加角色校验，防止前端绕过
   - 特别校验：状态 14→15/16 的推进必须由总部风控角色触发（如 D365 内存在审批入口）
