### 3. CofaceIntegrationDataSyncPlugin transaction报错修复 ✅

**错误**：`ISV code reduced the open transaction count`

**根因**：Plugin中catch了 `service.Create/Update/RetrieveMultiple` 异常后继续执行其他OrganizationService操作，导致D365事务计数不匹配。

**修复**（3处）：
1. `WriteToCustomerTags` — 移除foreach内吞 `service.Create/Update` 异常的try-catch
2. `GetScoringCardItems` — 移除吞 `service.RetrieveMultiple` 异常的try-catch
3. 主catch块 — 移除事务回滚时再次调用 `service.Update(errorRecord)` 的代码

**核心原则**：Plugin中不应catch OrganizationService调用异常后继续执行其他OrganizationService操作。

### 4. CustomerTagValidationPlugin depth拦截修复 ✅

**错误**：Coface数据集成阶段（状态11）写标签被拦截，提示"只有在人工复核阶段才能修改"

**根因**：`CustomerTagValidationPlugin` 在 `mcs_customer_tag` Update PreOperation 强制要求关联creditRecord状态为12。

**修复**：增加 `context.Depth > 1` 跳过校验，允许Plugin内部调用时写入标签。

## 关键经验速记

### Plugin事务异常
```
ISV code reduced the open transaction count
→ 不要catch service.Create/Update/Retrieve异常后继续执行service操作
→ 应让异常向上传播，由D365统一回滚
```

### DLL上传超时
```
8MB+ DLL命令行上传易超时
→ register-plugin-advanced × 1（上传DLL）
→ register-step-only × N（批量注册Step，每个2-5秒）
```

### Git推送失败
```
fatal: unable to access... Connection reset
→ 检查git credential-manager缓存是否过期
→ git credential-manager erase 清除后重试
→ 或检查代理配置
```
