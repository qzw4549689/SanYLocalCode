# Backups 备份文件夹

> **用途：** 项目文件的备份归档  
> **规则：** 按模块/类别分子文件夹，备份文件名必须包含时间戳

---

## 目录结构

```
Backups/
├── README.md              # 本说明文件
├── Planning/              # 开发计划备份
├── Tests/                 # 测试用例备份
├── BugReports/            # Bug记录备份
├── DailyReports/          # 日报备份
├── Documents/             # 其他文档备份
├── Code/                  # 代码备份
└── ...                    # 按需新增
```

## 备份命名规范

```
<原始文件名>_<YYYYMMDD>_<HHMMSS>.<扩展名>
```

**示例：**

| 原始文件 | 备份文件名 |
|---------|-----------|
| `开发计划.md` | `开发计划_20260606_115430.md` |
| `测试用例总集.md` | `测试用例总集_20260606_115430.md` |
| `测试Bug反馈记录.md` | `测试Bug反馈记录_20260606_115430.md` |
| `工作日报汇总.md` | `工作日报汇总_20260606_115430.md` |

## 备份操作示例

```bash
# 1. 进入对应备份子文件夹（如不存在则创建）
cd Backups/Planning

# 2. 复制文件并加上时间戳
cp ../../Documents/Planning/开发计划.md "开发计划_$(date +%Y%m%d_%H%M%S).md"
```

## 注意事项

- 每次重大修改前建议先备份
- 备份文件不要直接编辑，需要修改时复制回原始位置
- 定期清理过期的旧备份（保留最近 3-5 个版本即可）
